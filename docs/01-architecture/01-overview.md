## Протокол обмена (WebSocket)

Всё общение между Local Proxy и NestJS идёт через **одно** WebSocket-соединение.
Разные SOCKS5-сессии мультиплексируются через поле `id` (UUID).

```
Формат сообщения: { event: string, data: { id: string, ... } }
```

| Направление | event | data | Описание |
|-------------|-------|------|----------|
| → Сервер | `connect` | `{ id, dstAddr, dstPort, atyp }` | Открыть TCP соединение |
| → Клиент | `connect_resp` | `{ id, success, error? }` | Результат соединения |
| ↔ Любая | `data` | `{ id, data }` | Потоковые данные (base64) |
| ↔ Любая | `close` | `{ id }` | Закрыть сессию |
| → Сервер | `udp_associate` | `{ id }` | Запросить UDP релей |
| → Клиент | `udp_associate_resp` | `{ id, success, relayPort? }` | Результат UDP |
| ↔ Любая | `udp_data` | `{ id, data, dstAddr, dstPort, srcAddr, srcPort }` | UDP датаграмма |

## Типы адресов (ATYP)

| Код | Тип | Пример |
|-----|-----|--------|
| `0x01` | IPv4 | `93.184.216.34` |
| `0x03` | Доменное имя | `example.com` |
| `0x04` | IPv6 | `2606:4700::6810:85e` |

## Ключевые решения

| Решение | Почему |
|---------|--------|
| Одно WS-соединение на все сессии | Меньше overhead, не нужно переподключаться для каждого CONNECT |
| Base64 в data | WS фреймы — строки, бинарные данные безопасно кодируются |
| connectionId (UUID) | Глобально уникальный, не пересекается между разными клиентами |
| ATYP=0x03 поддерживается | Браузеры отправляют домены, а не IP (DNS на стороне прокси) |
| @nestjs/platform-ws | Нативный ws, без лишней обвязки Socket.IO |
