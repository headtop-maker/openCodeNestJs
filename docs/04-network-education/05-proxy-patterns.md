# Паттерны прокси

## Forward Proxy (прямой прокси)

То, что мы сделали в этом проекте. Клиент явно настраивает прокси в браузере,
весь трафик идёт через прокси-сервер.

```
Browser → [PROXY] → Internet
          ┌──────┐
          │SOCKS5│
          └──────┘
```

Плюс: контроль трафика, обход блокировок.
Минус: нужно настраивать каждый клиент.

## Reverse Proxy (обратный прокси)

Сервер стоит перед реальным сервером, принимает трафик от клиентов и
проксирует его на внутренние серверы.

```
Internet → [REVERSE PROXY] → Web Server
           ┌───────┐
           │ Caddy │ → Node.js App
           │ nginx │
           └───────┘
```

**В проекте:** Caddy в `docker-compose.yml` работает как reverse proxy:
он принимает HTTPS (443), расшифровывает и пробрасывает на NestJS (8080).

Плюс: SSL termination, балансировка, кэширование.
Минус: дополнительный hop.

## Tunnel Proxy (туннель)

Прокси создаёт туннель от клиента до целевого сервера.
Данные передаются как есть, без инспекции.

```
Browser → [TUNNEL] → Target Server
          ┌───────┐
          │  WS   │─── TCP ──► google.com
          └───────┘
```

Наш проект — именно туннель. Мы не разбираем HTTP, не модифицируем данные,
просто передаём байты туда-сюда.

## SOCKS5 vs HTTP-прокси

| SOCKS5 | HTTP-прокси |
|--------|-------------|
| Работает с любым протоколом (TCP/UDP) | Только HTTP(S) |
| Не инспектирует трафик | Может изменять заголовки |
| Нет кэширования | Есть кэширование |
| Проще реализовать туннель | Сложнее, но больше возможностей |

SOCKS5 — выбор для проектов типа «просто передай байты».

## Proxy Chain (цепочка прокси)

```
Browser → SOCKS5 :1080 → WS Tunnel → NestJS → SOCKS5 :9050 → Tor → Internet
```

Каждый прокси в цепочке добавляет свой уровень анонимности/контроля.
Наш туннель может быть частью такой цепочки.

## Наша схема — какой это паттерн?

```
Browser → [Forward SOCKS5 Proxy] → [WebSocket Tunnel] → Target
           ▲                    ▲                      ▲
           │  Явная настройка   │  Прозрачный туннель  │
           └────────────────────┘  через WS            └── Прямое TCP
```

Гибрид:
1. **Forward proxy** (SOCKS5) — пользователь явно настраивает браузер
2. **Tunnel** (WS + TCP) — данные передаются без изменений
